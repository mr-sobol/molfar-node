# Програмний модуль "molfar-node" | Вступ

Програмний модуль molfar-node (opens new window) – "Програмний модуль контейнера мікросервісів" - це RESTfull сервіс, написаний мовою програмування JavaScript, який надає можливості для розгортання та управління мікросервісами, такі як: запуск, налаштування, завершення та згортання мікросервісів.

Функціональне призначення

Програмний модуль призначений для розгортання (завантаження) та управління мікросервісами, зокрема для: запуску, налаштування (конфігурування), завершення та згортання (вивантаження) мікросервісів.

Опис логічної структури

Програмний модуль складається з:

  • javascript – програмний підмодуль, що приймає та оброблює запити

Використовувані технічні засоби

Програмний модуль експлуатується на сервері під управлінням Node.js, а також HTTP-клієнт axios на основі промісів для нього. В основі управління викликів з теміналу є менеджер пакетів npm. В основі управління всіх сервісів є система оркестрації Kubernetes, де всі контейнери працюють з використанням Docker.

Виклик та завантаження

Для запуску програмного модуля molfar-node необхідно:

  1. Клонувати репозиторій:
git clone https://github.com/wdc-molfar/molfar-node.git
  1. Перейти в директорію склонованого репозиторія та проінсталювати всі залежності:
cd /molfar-node
npm install

Завантаження програмного модуля забезпечується введенням в WEB-браузері адреси завантажувального модуля http://{hostname} (opens new window) з можливими вказівками:

  • / (opens new window) - метод GET для виклику сторінки із загальним описом сервісу
  • /state (opens new window) - метод GET для отримання інформації щодо стану та працездатності сервісу
  • /deploy (opens new window) метод POST – для завантаження сервісу в контейнер (розгортання сервісу) та повернення інформації про результат завантаження
  • /deploy/{id} (opens new window) метод POST – для завантаження сервісу в контейнер за ідентифікатором id (розгортання сервісу за ідентифікатором id) та повернення інформації про результат завантаження
  • /start (opens new window) метод POST – для запуску сервісу в контейнері та повернення інформації про результат запуску
  • /start/{id} (opens new window) метод POST – для запуску сервісу в контейнері за ідентифікатором id та повернення інформації про запуск
  • /config (opens new window) метод POST – для налаштування сервісу в контейнері (конфігурації сервісу) та повернення інформації про результат налаштування
  • /config/{id} (opens new window) метод POST – для налаштування сервісу в контейнері за ідентифікатором id (конфігурації сервісу за ідентифікатором id) та повернення інформації про результат налаштування
  • /terminate (opens new window) метод POST – для завершення сервісу в контейнері (зупинки сервісу) та повернення інформації про результат завершення
  • /terminate/{id} (opens new window) метод POST – для завершення сервісу в контейнері за ідентифікатором id (зупинки сервісу за ідентифікатором id) та повернення інформації про результат завершення
  • /undeploy (opens new window) метод POST – для вивантаження сервісу із контейнера (згортання сервісу) та повернення інформації про результат вивантаження
  • /undeploy/{id} (opens new window) метод POST – для вивантаження сервісу із контейнера за ідентифікатором id (розгортання сервісу за ідентифікатором id) та повернення інформації про результат вивантаження
